# SMAI 全画面 UI品質スプリント（3サイクル）

## 目的と前提

全画面の基本操作を、投資判断ロジックを変えずに検証・改善する反復用の計画です。通常回帰は外部通信に依存させず、`tests/fixtures/ui_sprint_mock_config.yaml` と `SMAI_DISABLE_BACKGROUND_WORKERS=1` を使います。Live provider、外部ニュース、LLM、通知配送は通常回帰から除外し、別の live smoke として扱います。

対象画面は、銘柄コックピット、銘柄ランキング、投資レーダー、Myウォッチリスト、SMAIアシスタント、リバランス、設定 / データ情報です。

## 第1週: 画面遷移と基本操作

| 因子 | 水準 | 操作・合格基準 |
| --- | --- | --- |
| 画面 | 7画面 | 各画面を開き、見出し、主要操作、例外なしを確認する。 |
| 主操作 | 画面ごとに1操作 | Cockpitのmock取得、Rankingの条件設定/作成、Radarの絞り込み/銘柄導線、Watchlistの表示切替/更新、Assistantの入力、Rebalanceの確認、Settingsの参照を確認する。 |
| 実行状態 | 初回 / セッション復元 | 選択画面・選択銘柄を跨いで不正な状態が残らない。 |

実行の中核は `tests/test_ui_three_week_sprint.py` です。画面ごとの初期表示に加え、リバランスの決定論的な「配分見直しを確認」操作からリスクサマリー表示までを実行します。Cockpit、Ranking、Radar、Watchlist、Assistantの詳細操作は各既存画面テストも併用します。

## 第2週: データ状態と失敗時の継続性

| 因子 | 水準 | 操作・合格基準 |
| --- | --- | --- |
| データ状態 | mock正常 / 空 / stale / 既存cache | 空を成功扱いせず、データ品質・鮮度・前回値を区別して表示する。 |
| 外部依存 | provider無効 / timeout・Gateway不通 | 主要画面が使え、例外や秘密情報を画面へ出さない。 |
| 更新対象 | スコアあり / featureなし | Watchlist更新が価格・利用可能なスコアを保存し、任意のfeature行の欠落だけで更新失敗にならない。 |

回帰対象は `tests/test_ui_forecast_display.py`、`tests/test_ui_news_streamlit_page.py`、`tests/test_ui_copilot_view.py`、`tests/test_ui_rebalance_app.py` です。失敗を再現した Watchlist の部分プレビューは、price と score 行だけを返す水準を追加の境界ケースとして扱います。

## 第3週: レスポンシブ・性能・操作性

| 因子 | 水準 | 操作・合格基準 |
| --- | --- | --- |
| viewport | 375x812 / 810x1080 / 1080x810 / 1366x768 | ページ全体の横はみ出し、切れた主要操作、Streamlit例外がない。 |
| 画面負荷 | 通常 / 大きい銘柄候補表 / 長いカード一覧 | 初期表示を過度に重くせず、表だけを局所スクロールにする。 |
| 操作性 | タップ / キーボード / ダイアログ | 主要ボタンは押下可能で、ダイアログと入力欄が viewport 内に収まる。 |

viewport の自動確認は、起動済みの隔離Streamlitに対して既存 `tests/ui/test_responsive_*.py` を `SMAI_RUN_RESPONSIVE_SMOKE=1` で実行します。対象viewportと手順は `docs/responsive/README.md` を正とします。通常CIには含めず、外部通信なしの隔離環境で実行します。

## 実行・改善の記録方法

各週は、(1) 因子と水準を固定、(2) network-free テストを実行、(3) 失敗を最小再現、(4) 影響範囲を限定して修正、(5) 失敗テストと対象画面テストを再実行、の順に進めます。報告時は成功数だけでなく、未実行の live smoke と実機ブラウザ確認を明示します。

## 2026-07-12 実行結果

- 第1週: `tests/test_ui_three_week_sprint.py` で7画面の初期表示、主要操作の存在、例外なしを確認し、リバランスの「配分見直しを確認」からリスクサマリーまでを実行しました。
- 第2週: Watchlist更新で、price と score 行は取得できるが任意の feature 行がないプレビューを再現しました。従来は属性参照エラーにより1件全体を失敗扱いにしていましたが、任意行を空配列として扱うよう修正し、同ケースを成功へ回復しました。Cockpit / Watchlistの上向き兆候表示と、Rankingの現行キャッシュ世代・指標ソート説明に合わせ、古い回帰期待も更新しました。
- 第3週: バックグラウンド処理を停止した隔離環境で、全画面スモーク8件は12.22秒で成功しました。追加の画面・ランキング回帰471件も成功し、Ruff と対象Black確認は成功しました。
- 未実行: 対話ブラウザ実体をこの実行環境で取得できなかったため、375 / 810 / 1080 / 1366px の実viewport操作、Safari、live provider、外部LLM・通知配送は実行していません。これらは上記の opt-in responsive/live smoke で別途確認します。

## 2026-07-12: 実画面フォローアップ（3スプリント完了）

- Sprint 1（銘柄確認）: 隔離したmock Streamlitで、銘柄コックピットの`7203.T`取得、Myウォッチリストへの追加、リバランスの`配分見直しを確認`、設定 / データ情報の確認までをPC実画面で操作しました。
- Sprint 2（探索・対話）: 投資レーダーの詳細フィルタとヒートマップ銘柄導線、銘柄ランキングの履歴・国／市場条件ダイアログ、SMAIアシスタントへの質問送信をPC実画面で操作しました。3本の横断ユーザーパスは `tests/ui/test_ui_user_paths_smoke.py` として回帰化しています。
- Sprint 3（配置・可読性）: Cockpit、Ranking、投資レーダー、Myウォッチリスト、Assistant、リバランス、設定 / データ情報を 375×812 / 810×1080 / 1080×810 / 1366×768 で実操作しました。全7画面のviewportスモークは **7 passed, 1 skipped**（任意チャート描画）、横断ユーザーパスは **3 passed** でした。
- 改善: タイポグラフィ縮小後もPC 36px・タッチ画面 44pxの操作高さを保証しました。固定ユーザー操作は本文と重ねず、サイドバー展開中は非表示にして閉じる操作を優先します。スマホの共通ヘッダーと本文開始余白を圧縮し、画面固有の見出し・主要操作を初期表示へ寄せました。
- 文言・配色: 重複した画面説明と過剰な安全注記をページ冒頭から整理し、根拠・不確実性が必要な詳細表示は維持しました。ベース配色と主要CTAを青／ネイビー基調に戻し、緑は上昇・成功などの状態表現に限定しました。
- 未実行: 実機Safari / PWA、live provider、外部LLM、実通知配送は、network-freeの通常回帰と分離したmanual / live smoke対象です。
