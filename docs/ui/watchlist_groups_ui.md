# ウォッチリストグループ UI設計

## 1. 設計原則

- グループは閉じたフォルダではなく、常時見えるセクションとする。
- 通常閲覧と配置編集を明確に分け、誤操作を避ける。
- D&Dが使えなくても全操作を完了できる。
- 現行の詳細dialog、Cockpit、判断メモ、解除を維持する。
- スコアは整理の参考表示であり、グループ名や配置を投資推奨に見せない。

## 2. 画面上部

```text
Myウォッチリスト
お気に入り銘柄を、投資テーマ・市場・検討状況ごとのグループに整理できます。

[＋ グループを作成] [配置を編集]  表示 [グループ別 ▼]
[↻ ウォッチリストを更新]
```

- PCでは作成・編集・表示を同じ操作列、更新を右寄せにする。
- 767px以下ではボタンを1列または2列にstackし、各ボタンを横幅いっぱいにする。
- defaultユーザーのsession-only説明は現行どおりタイトル直下に置く。
- お気に入りが0件でもグループ作成は許可する。既存の探索CTAも残す。

表示モード:

- `グループ別`: 新規既定。各セクションへ現行カードのcompact variantを配置する。
- `すべて`: 現行のカード／テーブル切替と全件表示を保つ。

初回導入時に既存利用者の情報密度を急変させないため、リリース判断で
`すべて`を移行期の既定にできる。実装前に既存session keyとの互換をテストする。

## 3. グループ別表示

```text
日本個別株  2件                                      [グループ編集]
────────────────────────────────────────────────────────
[compact card] [compact card]

未分類  1件
まだグループに配置していないお気に入り銘柄です。
────────────────────────────────────────────────────────
[compact card]
```

- group headerは名称、表示件数、編集ボタンを含む。
- 件数は現在のお気に入りかつ現在のフィルターに一致する表示件数とする。
- 0件のユーザーグループも表示し、空状態「まだ銘柄はありません」を出す。
- 未分類は最後。通常時に0件なら省略し、編集時は空section表示する。
- filter/sortは全体で1組とし、各グループ内へ同じ条件を適用する。
- グループを跨ぐAI総合順にはせず、group orderを優先する。

## 4. コンパクトカード

現行`_favorite_display_payload`をview modelとして再利用し、
現行`_favorite_card_html`とは別のcompact rendererを追加する。

表示:

- 銘柄名、symbol
- AI総合、上昇気配、下振れ警戒（未取得を許容）
- 値動き・更新状態の短いbadge
- `Cockpit画面で確認`
- 通常時の補助menuまたは詳細dialogから判断メモ・解除

グループ別で現行フルカードをそのまま3列反復すると縦長になるため避ける。
一方、既存payload、status tone、欠損表示、Cockpit handoffは再利用し、
評価値の意味を別実装にしない。

## 5. グループ作成dialog

`st.dialog("グループを作成")`を利用する。

- グループ名: `st.text_input(max_chars=32)`
- 説明: 任意、最大200文字
- トーン: cyan / blue / purple / green / amber / orange / rose / slate
- 文字数と最大20件を送信時にdomain/serviceでも再検証する。
- `作成`: primary CTA
- `キャンセル`: secondary
- 作成成功後はdialog stateを消し、1回だけrerunする。
- 同名・不正文字・保存失敗はdialog内に日本語で表示し、入力を保持する。

簡易プレビューで選択トーン、名称、説明、0件badgeを確認できる。

## 6. グループ編集dialog

- グループ名
- `上へ` / `下へ`
- `保存`
- `キャンセル`
- `グループを削除`

上下端では移動ボタンをdisabledにする。削除は同じ送信で即時実行せず、
確認dialogへ遷移する。

## 7. 削除確認dialog

```text
このグループを削除しますか？
中の銘柄はお気に入りから削除されず、「未分類」に移動します。

[削除する] [キャンセル]
```

- group名と対象件数を表示する。
- `削除する`はdanger、`キャンセル`はsecondary。
- 未分類に編集・削除入口を表示しない。
- 成功後に対象groupのdialog/draft keyを全て消す。

## 8. 配置編集モード

```text
配置編集モード
銘柄ごとに移動先を選び、「保存」で確定してください。
[編集を終了]

Nintendo / 7974.T
現在: 日本個別株
移動先 [日本個別株 ▼] [保存]
```

- `watchlist_groups_edit_mode`はsession stateに保持する。
- select keyはuser IDとnormalized symbolを含める。
- 選択肢のvalueは表示名でなく`group_id`、未分類は専用sentinelを使う。
- 保存成功時だけ現在表示を更新し、二重保存を避ける。
- `編集を終了`で未保存draftを破棄する旨を必要に応じて案内する。
- 通常カードの解除操作は編集モード中に隠すか、配置操作から離す。

## 9. D&D導入時のUI

- 編集モード中だけdrag handleを表示する。
- カード全体をdrag対象にせず、明示的なhandleとキーボード代替を持つ。
- drop先を枠線と背景で表示し、成功後にtoastと現在groupを更新する。
- タッチ端末は検証に合格するまでD&Dを無効化する。
- select＋保存は端末を問わず残す。
- component障害時はグループ表示を維持し、selectへfallbackする。

## 10. 空状態

| 状態 | 表示 |
| --- | --- |
| お気に入り0件 | 現行のランキング／投資レーダー探索CTA＋グループ作成 |
| グループ0件 | 全銘柄を未分類に表示＋作成CTA |
| ユーザーgroupが空 | 「まだ銘柄はありません。配置を編集から移動できます」 |
| filter結果0件 | 「現在の条件に一致する銘柄はありません」 |
| 未分類0件 | 通常時は省略、編集時は空section表示 |

## 11. ボタン配色・文字色

既存`ui/styles.py`のanchor＋`:has()`方式とresponsive classを再利用し、
ページ固有の重複CSSを増やさない。

| 種別 | 操作 | 推奨表現 |
| --- | --- | --- |
| Primary | ＋ グループを作成 | cyan〜blue背景、白文字 |
| Edit | 配置を編集 | violet〜blue背景または明るい同系枠、白文字 |
| Positive | 保存／作成 | green〜cyan背景、contrastを満たす文字 |
| Secondary | キャンセル／編集を終了 | dark背景、明るい文字、明瞭な枠 |
| Danger | 削除する | red〜orange背景または赤枠＋明るい赤文字 |
| Tertiary | グループ編集／上へ／下へ | cardと区別できるborder、文字ラベル |

- WCAG AA相当のcontrastを目標とする。
- hoverだけでなくfocus-visibleを表示する。
- disabledはopacityだけに頼らず、disabled属性・borderも併用する。
- アイコン単独にせず、日本語ラベルを付ける。

## 12. レスポンシブ

- smartphone: `max-width: 767px`
- tablet: `768px–1024px`
- desktop: `1025px以上`
- 1080px landscapeはcompact desktop

| viewport | group card列 | 操作 |
| --- | --- | --- |
| 375×812 | 1列 | CTA stack、selectと保存も縦積み |
| 810×1080 | 2列 | header操作はwrap可 |
| 1080×810 | 2〜3列 | 過密な4列を避ける |
| 1366×768以上 | 3列 | 現行情報密度を維持 |

ページ全体の横scrollを禁止する。長いグループ名は2行までまたはellipsis＋
accessible labelとし、select・dialog・buttonはviewportからはみ出さない。

## 13. 状態遷移

```text
通常表示
 ├─ 作成 → 作成dialog → 保存 → 通常表示
 ├─ group編集 → 編集dialog → 保存 → 通常表示
 │                         └→ 削除確認 → 削除 → 通常表示
 └─ 配置を編集 → 配置編集
                    ├─ select変更 → 保存 → 配置編集
                    └─ 編集を終了 → 通常表示
```

ユーザー切替は全状態から通常表示へ戻し、前ユーザーのgroup IDを持ち越さない。

## 14. エラー表示

- 入力エラー: 対象dialog/form内の`st.error`
- 保存競合・I/O失敗: 現在表示を維持し、再試行可能な`st.error`
- 破損ファイル: お気に入りは表示し、全件を未分類として安全にfallback
- 存在しないgroup参照: 未分類に表示し、必要なら非阻害warning
- D&D component障害: select fallbackへ切替

内部path、stack trace、生JSONは通常UIに表示しない。
