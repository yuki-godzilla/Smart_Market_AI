# SMAI レスポンシブ確認チェックリスト

対象 viewport: 375x812 / 810x1080 / 1080x810 / 1366x768

## 共通

- [ ] ページ全体に不要な横スクロールがない
- [ ] 主要ボタンが切れず、スマホで押しやすい
- [ ] 長い銘柄名、URL、数値がカード外にはみ出さない
- [ ] 表・ヒートマップだけ必要に応じて横スクロールできる
- [ ] ローディングとダイアログが viewport 外にはみ出さない
- [ ] Streamlit エラー表示がない
- [ ] PC の情報密度が過度に低下していない

## Cockpit

- [ ] 銘柄選択と更新 CTA がスマホで縦積みになる
- [ ] KPI / 価格カードが iPhone 1列、iPad 縦 2列で読める
- [ ] チャートが画面幅内に収まる
- [ ] AI予測、Research Summary、Decision Report CTA が切れない

## Ranking

- [ ] カードが iPhone 1列、iPad 縦 2列、PC 3列で読める
- [ ] フィルターと並び順がスマホで操作できる
- [ ] Cockpit 導線とお気に入り操作が切れない
- [ ] 詳細表がページ全体を押し広げない

## My Radar

- [ ] 監視銘柄カードと状態色がスマホで読める
- [ ] フィルター、更新、Cockpit 導線が押しやすい
- [ ] 再取得中の表示が viewport 内に収まる

## 投資レーダー

- [ ] 市場サマリーとニュースカードがスマホで読める
- [ ] 出典リンクが押しやすく、長い URL がはみ出さない
- [ ] ヒートマップのスクロールがコンポーネント内に限定される
- [ ] 更新・キャッシュ状態が切れない
- [ ] `今日のテーママップ`で、`市場データ`と`ニュース代理 / 方向未確認`の意味を色だけに頼らず読める
- [ ] 初期テーママップはPCで最大3テーマ・各最大3タイルを候補キュー直前に比較でき、残りテーマの展開操作と件数が分かる
- [ ] タイルの面積が重複除外済み根拠記事数、色が鮮度、`本文に出た` / `テーマ関連`が直接性であることを、色だけに頼らず読める
- [ ] iPad縦ではテーマ枠を1列にして、サイドバーと2列表示でタイルの企業名・根拠数が潰れない
- [ ] 3由来レーンの初期候補（各3〜4件、全体最大12件を目安）と`あと N 件を見る`がPC/iPad/iPhoneで操作できる
- [ ] iPadで候補一覧と詳細を狭い2列に固定せず、候補を下まで読んでも空白だけの詳細領域が残らない
- [ ] 選択候補の詳細ダイアログまたは直後の詳細から、`根拠資料を確認`と`銘柄コックピット`へ到達でき、閉じる操作もviewport内に収まる

## SMAI Assistant

- [ ] チャット本文と入力欄が画面内に収まる
- [ ] Tool Plan と確認ボタンが切れない
- [ ] 保存 / ZIP / Decision Report 操作が押しやすい
- [ ] 機能ロジックと確認フローが従来どおりである
