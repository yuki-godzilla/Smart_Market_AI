# 33 LLMランキング融合 戦略

## 1. 目的

通常ランキングの速度と安定性を維持し、ユーザーが明示的にONにした場合だけ、上位候補の材料確認をLLMで補助する。「本気分析モード（AI材料分析つき）」は買い推奨や全銘柄ランキングエンジンではない。

## 2. 初期スコープ

通常ランキングはLLM非依存で完走する。本気分析モードは完成後の上位10件についてResearch RAG、News、TDnet、IR、EDINET、provider profileまたはcacheを参照し、好悪材料、カタリスト、配当罠、ニュース関連度、材料鮮度、根拠品質、不確実性を構造化する。

初期段階では順位を変更せず、材料バッジ、警告、要約だけを表示する。LLM失敗時も通常ランキング結果を保持する。

## 3. データフロー

1. 数値モデルだけで通常ランキングを作成する。
2. 上位N件を抽出する。
3. 外部sourceを明示取得するかcacheを参照する。
4. LLM Material Assessmentを生成する。
5. バッジ、警告、要約を表示する。
6. 性能評価通過後だけ、上位候補内で軽い順位補正を検討する。

取得本文は既定で一時的に扱い、永続化は別の明示archive/save actionとする。source URL、`published_at`、`fetched_at`、`source_hash`、model name、prompt versionは評価・再現用metadataとして管理する。

## 4. 性能評価ゲート

同一条件でLLMなし/ありを比較し、false positive削減、配当罠・悪材料・無関係ニュースの検出、説明品質、latency、failure rate、cache hit rateを測る。採用、バッジ限定採用、不採用を判断し、寄与がなければCockpit、Decision Report、材料バッジに限定する。

## 5. Performance Budget

- 全銘柄へ同期的・リアルタイムにLLM分析しない。
- 初期対象は上位10件、local LLM workerは基本1並列。
- 高度予測とLLMの同時多重実行を制限する。
- source hash不変時は結果を再利用する。
- TTL目安はニュース24時間、IR/TDnet 3〜7日、決算・配当安全性7〜30日。
- LLM分析はランキング本体を失敗させない。

## 6. UIと完了条件

ランキング作成付近に初期値OFFで表示し、「上位候補についてニュース・開示・IRをAIで確認し、悪材料・好材料・配当罠を補助評価します。通常より時間がかかります。」と説明する。処理中、完了、部分失敗を区別し、買い推奨ではないことを明示する。

通常ランキングがLLMなしで完走し、明示ON時だけ上位候補を分析し、評価前は順位を変えず、失敗時も通常結果を残せることを完了条件とする。
